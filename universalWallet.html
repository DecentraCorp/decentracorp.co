The Universal Wallet

The Universal Wallet is an EVM(Ethereum Virtual Machine) compatible wallet built entirly from web3 technologies. Powered by IPFS(the Inter Planetary File System) and DecentraCorps Proof of Authority Network the Universal Wallet aims to bring the familiarity and convenience of Web2 to the power of Web3 by securly abstracting away a users public/private key pairs into a email/password like set-up!

Account Abstraction The Future of dApps

With Account Abstraction, your ethereum account is availible to you in any compatable dApp on ANY device. Access your Universal Wallet securly from your Desktop, smart-phone and tablet with just your email and password!

IPFS Integration Powered by the Decentralized Web

Your Universal Wallet Account is encrypted and uploaded to the Inter Planetary File System(IPFS) ensuring your wallet exists in many places at once. Once Created, Your account can be recalled and decrypted for use on any Device!

Universal Wallet Wrapper An Open-Source tool for dApp Devs

The Universal Wallet Wrapper is an NPM package aimed at making it easy for dApp developers to seamlessly integrate the Universal Wallet into their dApps, eliminating the need for users to manage which account they use on which device!

FAQ

What problem does the Universal Wallet Address?

A major hurtle to web3 and dApp adoption is what is known as the "user onboarding problem". The User onboarding problem refers to the dificulty a user faces when interacting with dApps as opposed to web2 based applications. Every day end users dont want to worry about account management like private key storage or importing their account accross multiple devices. Users Expect a web2 like experience where their email and password allow them to acces an application from ANY device.

How does the Universal Wallet address this problem?

The Universal Wallet Abstracts away a users public/private keypair using web3 and industry standard encryption technologies so that the end user only needs to keep track of which Email and password they use accross all universal Wallet powered dApps!

How does the Universal Wallet accomplish this?

Upon sign-up the Universal wallet uses the web3.js library to create a new public/private key pair. This keyPair is then encrypted into a standard JSON object using the users credentials. This JSON is then uploaded to IPFS which returns a hash representing the encrypted JSON file. This Hash is then encrypted again using the same credentials and uploaded to the Account Abstraction Contract on DecentraCorps Proof of Authority Network where it is linked to the users Email. This set-up allows for a heavily encrypted account key file to be called from any device using the users email address where it can then be decrypted and used by the front end of a dApp.

What types of Accounts are supported?

At the moment the Universal Wallet only supports EVM compatible accounts. This means that one account can be created with the Universal Wallet and used accross all EVM based networks like the Ethereum Mainnet, ALL Ethereum testnets, Ethereum Classic's Mainnet, the DecentraCorp Proof of Authority network and MANY More. In the future we plan to add support for other networks and tokens like Bitcoin, Litecoin and Bitcoin Cash.

Who can use the Universal Wallet?

The Universal Wallet is Open-Source and free to implement/use. This means that ANY dApp developer looking to give their users a better user experience and or implement universal account access accross all devices can freely implement the Universal Wallet into their dApps. We are starting construction of a React compatible wrapper first, then moving to other popular frameworks like angular and vue. From here we plan to implement a python version followed by a java version so that dApp developers of any language can curate a better UI/UX for their end users!
